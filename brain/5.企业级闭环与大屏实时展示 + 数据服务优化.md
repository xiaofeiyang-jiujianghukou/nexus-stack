# 三、核心模块设计

## 1️⃣ 数据服务层（SpringBoot + Redis）

* **目的**：减少大屏请求 ClickHouse 压力
* **思路**：

    * Flink SQL → ClickHouse 写入实时指标表
    * SpringBoot 定时拉取/监听更新
    * 缓存到 Redis
    * 前端请求直接走 Redis

---

### 核心接口设计

```java
@RestController
@RequestMapping("/api/dashboard")
public class DashboardController {

    @Autowired
    private StringRedisTemplate redisTemplate;

    @GetMapping("/gmv")
    public Double getGmv() {
        String value = redisTemplate.opsForValue().get("gmv_last_10s");
        return value != null ? Double.valueOf(value) : 0;
    }

    @GetMapping("/user-level")
    public Map<String, Integer> getUserLevel() {
        String json = redisTemplate.opsForValue().get("user_level_count");
        return json != null ? parseJson(json) : Map.of();
    }
}
```
