---

## 2️⃣ 定义 Broadcast State

```java
MapStateDescriptor<Long, User> userStateDesc =
        new MapStateDescriptor<>(
                "user-state",
                Long.class,
                User.class
        );
```

---

## 3️⃣ 构建流

```java
DataStream<Order> orderStream = ...
DataStream<User> userStream = ...
```

---

## 4️⃣ 用户流 → 广播

```java
BroadcastStream<User> userBroadcast =
        userStream.broadcast(userStateDesc);
```

---

## 5️⃣ 订单流 JOIN 用户

```java
SingleOutputStreamOperator<WideOrder> wideStream =
        orderStream
                .keyBy(Order::getUserId)
                .connect(userBroadcast)
                .process(new BroadcastProcessFunction<Order, User, WideOrder>() {

                    @Override
                    public void processElement(
                            Order order,
                            ReadOnlyContext ctx,
                            Collector<WideOrder> out) throws Exception {

                        User user = ctx
                                .getBroadcastState(userStateDesc)
                                .get(order.getUserId());

                        if (user != null) {
                            WideOrder wide = new WideOrder();
                            wide.setOrderId(order.getOrderId());
                            wide.setUserId(order.getUserId());
                            wide.setAmount(order.getAmount());
                            wide.setLevel(user.getLevel());

                            out.collect(wide);
                        }
                    }

                    @Override
                    public void processBroadcastElement(
                            User user,
                            Context ctx,
                            Collector<WideOrder> out) throws Exception {

                        ctx.getBroadcastState(userStateDesc)
                                .put(user.getUserId(), user);
                    }
                });
```

# 十、你必须理解的核心（面试重点）

直接帮你总结一段话术：

---

> 在实时数仓中，用户维度数据通常作为维表，通过 Broadcast State 进行全局广播，订单流作为主流进行关联。这种方式避免了双流Join的状态膨胀问题，同时保证了低延迟和高吞吐，是实时宽表构建的主流方案。
